Storage Engine

Every layer built from scratch.

No ORM. No framework. No third-party libraries. Just C11 and POSIX — every byte accounted for.

🗂️

Slotted Pages

8KB pages with 40-byte headers. Slot directory grows downward, record data grows upward from the end. Same design as PostgreSQL's bufpage.h. Variable-length keys and values with no external fragmentation.

🌳

B+Tree with Copy-on-Write

All data lives in leaf nodes chained rightward for efficient range scans. Mutations clone the path from root to leaf — old pages stay accessible to concurrent readers. MVCC snapshots without a version chain.

📝

Write-Ahead Log

Every mutation hits the WAL before the B+Tree. CRC-32C checksums on every record. Two-pass recovery replays only committed transactions. Corrupt records truncate the log cleanly on detection.

🔄

MVCC Transactions

Snapshot isolation by default. Serializable mode with read-set conflict detection. Write-set buffering with atomic commit via WAL sync. Readers never block writers — writers never block readers.

🧠

Full SQL Engine

Hand-written lexer recognising 60+ SQL keywords. Recursive-descent parser using Pratt precedence climbing for expressions. Supports SELECT, INSERT, UPDATE, DELETE, CREATE TABLE, DROP TABLE, transactions.

🌐

TCP Wire Protocol

Text-framed protocol — debuggable with netcat. epoll event loop on Linux with poll(2) fallback. JSON result payloads. Non-blocking I/O, TCP_NODELAY, 64KB per-connection buffers, graceful shutdown.

📦

Embeddable Library

Link libnovadb directly into your C application. Clean public API. Arena and slab allocators for zero-leak query execution. Thread-safe — share one engine handle across threads.

🛡️

Crash-Safe by Default

fsync after every commit. CRC-protected WAL. Buffer pool flushes all dirty pages on shutdown. Clock-sweep eviction keeps hot pages in memory. Panic on unrecoverable corruption — never silently lose data.

⚡

Clock-Sweep Eviction

O(1) page eviction with a single reference bit per cached frame. The clock hand sweeps through; if a page hasn't been touched, it's evicted. Performs nearly identically to LRU at a fraction of the overhead.

Getting Started

One command. That's it.

Clone, build, run. No Docker required. No package manager. No configuration files.

terminal — one command setup
# Run this single command:
curl -fsSL https://raw.githubusercontent.com/Saff9/NovaDB/main/setup.sh | sh

# Or manually:
git clone https://github.com/Saff9/NovaDB.git
cd NovaDB
./setup.sh

The setup script auto-detects your platform, checks for a C compiler, builds everything, runs the test suite, and prints instructions. Works on Linux, macOS, and FreeBSD.

Source Manual build

terminal
# Clone and compile
git clone https://github.com/Saff9/NovaDB.git
cd NovaDB
make release

# Start the server
./bin/novadb-server --data-dir ./mydata --port 9876

Docker docker compose up

terminal
# One Docker command:
docker compose up -d

# Connects on port 9876
# Data persisted in a Docker volume
# Includes healthcheck

Embed As a C library

terminal
# Link directly into your app:
cc -Iinclude -Isrc your_app.c \
   src/engine.c src/common/*.c \
   src/storage/*.c src/txn/*.c \
   src/sql/*.c src/network/*.c \
   -lpthread -lm -o your_app

Try some SQL

novadb-cli
./bin/novadb-cli

novadb> CREATE TABLE users (
  id INTEGER PRIMARY KEY,
  name VARCHAR(255)
);

novadb> INSERT INTO users VALUES (1, 'Alice');
novadb> SELECT * FROM users;

Page Layout

8KB slotted page — 40-byte header, slot directory from the top, record data from the bottom.

┌──────────────────────────────┐
│  Header (40B)               │
│  magic type flags self       │
│  parent sibling nslots       │
│  free_off free_len           │
├──────────────────────────────┤
│  Slot 0 Slot 1 ... Slot N   │← offset 40
│   off  kl  vl  fl           │  grows ↓
├──────────────────────────────┤
│       free space             │
├──────────────────────────────┤
│  Data N ... Data 1 Data 0 │← grows ↑
└──────────────────────────────┘

WAL Record

Every record is CRC-32C protected. Committed transactions are replayed on recovery.

Off  Sz  Field
  0   4  CRC-32C
  4   4  total_len
  8   8  LSN
 16   4  txn_id
 20   1  op (INSERT=1 DELETE=2
        COMMIT=4)
 21   2  key_length
 23   4  value_length
 27   -  key ...
27+kl -  value ...

Community

Help build the future of NovaDB.

Apache 2.0. All contributions are welcome — code, docs, bindings, benchmarks.

🔍

Pick an Issue

Browse the GitHub issue tracker. Look for good-first-issue or help-wanted. Comment to let others know you're working on it.

🔀

Fork & Branch

Fork the repo, branch off main. Name your branch feature/short-desc or fix/issue-42. Keep each branch focused on one change.

📐

Follow the Style

C11, 4-space indentation, no tabs. Functions are snake_case, types are CamelCase. Every function gets a brief comment. -Wall -Wextra -Wpedantic — zero warnings required.

🧪

Write Tests

New features need integration tests in test/. Bug fixes need a regression test that fails without your patch. Run make test before submitting.

📬

Open a Pull Request

Push your branch and open a PR. Describe what you changed and why. Link any related issues. CI runs on every PR automatically.

💬

Code Review

Maintainers review within a few days. Feedback is about the code, not the person. Iterate, learn, ship.
